1. The formulas (what is being measured)

1.1 Label generation — the “ground truth” (a performance proxy)

The label is a deterministic 5-class call, computed from future price (it is a training target, never a model input). Source: compare_lab/labeling.py.

ema      = EMA(price, span=3)
for tau in [3, 7, 15] trading days, weight w in [0.3, 0.5, 0.2]:
    r_tau  = (ema[t+tau] - ema[t]) / ema[t]      # FORWARD return (looks ahead)
    signal += w * ( r_tau / rolling_std_20(r_tau) )   # Sharpe-like, vol-normalised
label[t] = quantile_cut(signal, [0.03, 0.15, 0.53, 0.85])
           -> STRONG_SELL / SELL / HOLD / BUY / STRONG_BUY
Max look-ahead = 15 trading days (~3 weeks). The quantile cuts are asymmetric, reproducing the paper's bull-skewed class mix (3 / 12 / 38 / 32 / 15 %). The continuous signal before the cut is the label score used for IC below.

1.2 Decision matrix — the training reward (asymmetric)

How a predicted class is scored against the true label during RL (compare_lab/grpo/rewards.py). False-bullish (long into a crash) is penalised hardest; HOLD when action is needed is penalised; an unparseable reply scores −2.5.

pred \ truetrue SSELLtrue SELLtrue HOLDtrue BUYtrue SBUY
pred SSELL+1.00+0.75-1.25-2.00-2.00
pred SELL+0.75+1.00-0.75-1.50-2.00
pred HOLD-1.50-1.00+1.00-1.00-1.50
pred BUY-1.75-1.25-0.75+1.00+0.75
pred SBUY-2.25-1.50-1.25+0.75+1.00

1.3 Backtest metrics

  • CR — Cumulative Return = Π(1+r)−1
  • SR — Sharpe = annualised mean excess return / std (rf 4%/yr, √252)
  • HR — Hit Rate (ours = fraction of positive-return days; the paper's HR is directional accuracy — not directly comparable)
  • MDD — Max Drawdown

1.4 IC — label-fidelity (does the model learn the label score?)

Rank correlation (Spearman) between the model's predicted class (STRONG_SELL=0 … STRONG_BUY=4) and the actual forward label score (§1.1). 0 = chance, 1 = perfect. tail-weighted IC up-weights the extremes (thin tails of the asymmetric cut). Source: compare_lab/eval_labels.py.

4. The portfolio backtest (qf-lib engine)

The headline portfolio results run through the real qf-lib event-driven engine (compare_lab/backtest.py): a qf-lib AlphaModel (LONG held / OUT otherwise), FixedPortfolioPercentagePositionSizer at 1/8 = 12.5% each, IBCommissionModel commissions, daily order events, next-bar execution.

Position rule. Each ticker → 5-class decision → BUY/STRONG_BUY = held. Held names get equal weight (12.5%), capped at 8 slots (ranked by class strength if more are flagged); unused budget stays in cash. So it is not a fixed all-names equal-weight — it is “equal-weight the model's top-8 buy picks, rest cash.”

6. Caveats

  • Prediction ≠ profit. The highest-IC models lose money in flat 2025-H1; the keeper (lowest IC) makes the most in a bull run — its return is a bullish lean, not skill.
  • Bull-window flattery. The paper window is a bull run (buy&hold Sharpe 1.08); both our and the paper's high Sharpes partly come for free from being long.
  • Short window, annualised. 3 months annualised → high-variance Sharpes (a single good quarter).
  • Engine mismatch. §3 per-ticker = lightweight pandas (no commissions); §4 portfolio = qf-lib (with commissions).
  • HR not comparable (different definition).
  • Small model, thin input. Qwen3-4B on price-only/~2k-token snapshots vs the paper's larger model on 15–23k-token full multimodal.
